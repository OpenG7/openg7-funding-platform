# ProductionLaunchAgent

ProductionLaunchAgent is a semi-autonomous DevOps/SRE agent for the OpenG7
production VPS.

It can read YAML checklists, select typed tools, execute only whitelisted SSH
commands, store history in SQLite, and generate Markdown/JSON reports.

## Safety Model

- No arbitrary shell command execution.
- All remote commands are generated by `CommandRegistry`.
- Dangerous fragments are rejected: `rm -rf`, `shutdown`, `reboot`, `mkfs`,
  `fdisk`, `kill -9`, `iptables`.
- RBAC gates tools:
  - `viewer`: checks, logs, reports.
  - `operator`: deploy, backup, restart.
  - `admin`: rollback.
- LLM output can only select known tools. It cannot inject shell commands.
- Log output is treated as untrusted data.

## Environment

```bash
PLA_ROLE=viewer
PLA_DOMAIN=openg7.org
PLA_HEALTH_PATH=/health
PLA_APP_DIR=/opt/openg7-funding-platform
PLA_SSH_HOST=vps-8db0cb49.vps.ovh.ca
PLA_SSH_USER=ubuntu
PLA_SSH_PORT=22
PLA_PRIVATE_KEY_PATH=/path/to/private/key
PLA_DATABASE_PATH=apps/production-launch-agent/database/production-launch-agent.sqlite
PLA_REPORT_DIR=apps/production-launch-agent/logs/reports
OPENAI_API_KEY=
```

## Install

```bash
corepack enable
corepack yarn install
corepack yarn workspace @openg7/production-launch-agent build
```

## Dry Run

Dry run is the default. It validates the workflow without connecting over SSH.

```bash
corepack yarn workspace @openg7/production-launch-agent start
```

Use another checklist:

```bash
corepack yarn workspace @openg7/production-launch-agent start -- \
  --checklist apps/production-launch-agent/checklists/maintenance.yaml
```

Ask the LLM planner for a tool plan:

```bash
corepack yarn workspace @openg7/production-launch-agent start -- \
  --request "Vérifie le serveur et les logs API, sans déployer."
```

## Execute Against VPS

```bash
PLA_ROLE=operator \
PLA_PRIVATE_KEY_PATH=~/.ssh/openg7_vps \
corepack yarn workspace @openg7/production-launch-agent start -- --execute
```

Rollback requires admin:

```bash
PLA_ROLE=admin \
PLA_PRIVATE_KEY_PATH=~/.ssh/openg7_vps \
corepack yarn workspace @openg7/production-launch-agent start -- \
  --checklist apps/production-launch-agent/checklists/rollback-checklist.yaml \
  --execute
```

## Docker

```bash
mkdir -p apps/production-launch-agent/secrets
cp ~/.ssh/openg7_vps apps/production-launch-agent/secrets/production_launch_ssh_key
chmod 600 apps/production-launch-agent/secrets/production_launch_ssh_key

cd apps/production-launch-agent
PLA_ROLE=operator docker compose up --build
```

## Reports

Reports are written to:

```text
apps/production-launch-agent/logs/reports
```

Each run generates:

- `*.json`
- `*.md`

## Checklists

Available checklists:

- `checklists/production-launch-checklist.yaml`
- `checklists/rollback-checklist.yaml`
- `checklists/maintenance.yaml`

Checklist steps may be simple:

```yaml
steps:
  - check_disk
  - check_memory
```

Or parameterized:

```yaml
steps:
  - tool: analyze_logs
    params:
      service: api
```
